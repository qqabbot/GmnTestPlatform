# 自动化测试平台 - 前端设计方案

## 一、功能模块

### 1. 测试用例管理
- **用例列表**：展示所有测试用例，支持按模块筛选
- **用例创建/编辑**：表单式编辑器，支持配置请求参数、断言脚本等
- **用例删除**：批量删除和单个删除

### 2. 测试执行
- **执行配置**：选择环境（dev/staging/prod）、选择模块
- **实时执行**：显示执行进度和实时日志
- **执行历史**：查看历史执行记录

### 3. 测试报告
- **结果展示**：测试通过率、失败用例详情
- **可视化图表**：饼图、柱状图展示测试结果
- **导出报告**：支持导出 HTML/PDF 格式

## 二、页面结构

### 主布局
```
┌─────────────────────────────────────────┐
│  Header (Logo + 导航 + 用户信息)         │
├──────┬──────────────────────────────────┤
│      │                                  │
│ 侧边 │        主内容区                  │
│ 导航 │                                  │
│      │                                  │
│      │                                  │
└──────┴──────────────────────────────────┘
```

### 核心页面

#### 1. 用例列表页 (Test Cases)
- **顶部操作栏**：新建用例、批量删除、模块筛选
- **表格展示**：
  - 列：ID、用例名称、HTTP方法、URL、所属模块、状态、操作
  - 操作：编辑、删除、执行单个用例
- **分页器**

#### 2. 用例编辑器 (Case Editor)
- **基本信息**：
  - 用例名称
  - 所属模块（下拉选择）
  - HTTP 方法（GET/POST/PUT/DELETE）
  - 请求 URL
- **请求配置**：
  - 请求体（JSON 编辑器）
  - 前置条件（依赖的用例 ID）
- **脚本配置**：
  - 变量提取脚本（Groovy）
  - 断言脚本（Groovy）
- **底部操作**：保存、取消、测试运行

#### 3. 测试执行页 (Test Execution)
- **执行配置面板**：
  - 环境选择（dev/staging/prod）
  - 模块选择（可选）
  - 执行按钮
- **执行状态**：
  - 进度条
  - 实时日志流
  - 当前执行的用例名称
- **快速统计**：
  - 总用例数
  - 已执行数
  - 通过数
  - 失败数

#### 4. 测试报告页 (Test Report)
- **概览卡片**：
  - 通过率
  - 总用例数
  - 执行时长
- **结果图表**：
  - 饼图：通过/失败/跳过比例
  - 柱状图：各模块通过率
- **详细结果表格**：
  - 用例名称、状态、执行时长、错误信息
  - 支持展开查看详细日志

## 三、技术栈

### 前端框架
- **Vue 3** + **Composition API**
- **Vue Router** - 路由管理
- **Pinia** - 状态管理

### UI 组件库
- **Element Plus** - 企业级 UI 组件库
  - 表格、表单、对话框等
  - 图标库

### 代码编辑器
- **Monaco Editor** - 用于编辑 Groovy 脚本和 JSON

### 图表库
- **ECharts** - 数据可视化

### HTTP 客户端
- **Axios** - API 请求

### 工具库
- **Day.js** - 日期处理
- **Lodash** - 工具函数

## 五、交互设计

### 用例编辑
1. 点击"新建用例"打开抽屉式编辑器
2. 表单验证：必填项、URL 格式、JSON 格式
3. 保存成功后显示提示消息

### 测试执行
1. 选择环境和模块后点击"开始执行"
2. 显示加载动画和进度条
3. 实时更新执行状态
4. 完成后自动跳转到报告页

### 结果查看
1. 点击用例可展开详细信息
2. 失败用例高亮显示
3. 支持筛选和搜索

- **平板端**：768px-1199px，侧边栏可折叠
