# Test Scenario 使用指南 🚀

恭喜您！您的测试平台现已支持强大的**场景编排 (Test Scenario)** 功能。相比于旧版的 Test Plan，新功能支持复杂的流程控制（循环、判断、延时）和无限层级嵌套。

## 1. 快速入门
### 1.1 创建场景
1. 点击左侧菜单 **"Test Scenarios"** (原 Test Plans)。
2. 点击右上角 **"New Scenario"** 按钮。
3. 输入场景名称和所属项目，点击创建。
4. 系统将自动跳转到**场景编排编辑器**。

### 1.2 编排界面 (编辑器)
编辑器分为三个区域：
*   **左侧 (组件库)**: 
    *   **Logic Nodes**: 提供 `Group` (分组), `Loop` (循环), `If` (条件), `Wait` (等待) 等控制节点。
    *   **Test Cases**: 列出当前项目下的所有 API 测试用例。
*   **中间 (画布)**: 
    *   **拖拽**: 从左侧将节点拖入中间画布。
    *   **嵌套**: 您可以将用例拖入 Loop 或 If 节点内部，形成嵌套结构。
    *   **排序**: 上下拖拽调整执行顺序。
*   **右侧 (属性面板)**:
    *   点击画布中的任一节点，右侧会显示其属性。
    *   例如点击 `Loop` 节点，可以在右侧设置 `Loop Count` (循环次数)。

## 2. 核心功能详解
### 循环控制 (Loop)
*   **用途**: 重复执行某组步骤。
*   **配置**: 选中 Loop 节点，在右侧设置 `Count` (如 5，表示循环执行 5 次)。

### 条件控制 (If)
*   **用途**: 根据条件决定是否执行某组步骤。
*   **配置**: 支持 Groovy 表达式。例如：
    *   `${status} == 'PASS'` (如果之前的变量 status 为 PASS 则执行)
    *   `${response.code} == 200`

### 延时等待 (Wait)
*   **用途**: 在步骤间暂停一段时间。
*   **配置**: 设置 `Wait Time (ms)` (如 1000 表示等等 1 秒)。

## 3. 保存与执行
1.  **保存**: 编排完成后，点击画布右上角的 **"Save"** 按钮保存当前结构。
2.  **运行**: 点击 **"Run"** 按钮，系统将按顺序（或逻辑）执行所有步骤，并弹窗显示简要执行结果。
